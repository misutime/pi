# Pi 启动信息源码位置

启动 `pi` 时终端输出的信息定义在两个主要位置，均在同一文件中。

## 核心文件

**`packages/coding-agent/src/modes/interactive/interactive-mode.ts`**

---

## 1. Logo、快捷键提示、引导文字（约第 754–791 行）

```ts
const logo = theme.bold(theme.fg("accent", APP_NAME)) + theme.fg("dim", ` v${this.version}`);
```

输出: `pi v6.1.0`（带颜色）

### 紧凑模式快捷键行

```ts
const compactInstructions = [
    hint("app.interrupt", "interrupt"),
    rawKeyHint(`${keyText("app.clear")}/${keyText("app.exit")}`, "clear/exit"),
    rawKeyHint("/", "commands"),
    rawKeyHint("!", "bash"),
    hint("app.tools.expand", "more"),
].join(theme.fg("muted", " · "));
```

输出: `escape interrupt · ctrl+c/ctrl+d clear/exit · / commands · ! bash · ctrl+o more`

### 提示按 ctrl+o 查看更多

```ts
const compactOnboarding = theme.fg("dim",
    `Press ${keyText("app.tools.expand")} to show full startup help and loaded resources.`);
```

输出: `Press ctrl+o to show full startup help and loaded resources.`

### 引导文字

```ts
const onboarding = theme.fg("dim",
    `Pi can explain its own features and look up its docs. Ask it how to use or extend Pi.`);
```

输出: `Pi can explain its own features and look up its docs. Ask it how to use or extend Pi.`

### 展开模式的完整快捷键列表

按 `ctrl+o` 后展开显示，由 `expandedInstructions` 数组构建，包含所有快捷键说明。

## 2. `[Context]` / `[Skills]` / `[Prompts]` / `[Extensions]` / `[Tools]` 资源列表

在 `showLoadedResources()` 方法中生成（第 1417 行开始）。

### 节标题

```ts
const sectionHeader = (name: string, color: ThemeColor = "mdHeading") =>
    theme.fg(color, `[${name}]`);
```

依次调用 `addLoadedSection()` 生成五节：

| 节标题 | 数据来源 |
|--------|----------|
| `[Context]` | `this.session.resourceLoader.getAgentsFiles().agentsFiles` |
| `[Skills]` | `this.session.resourceLoader.getSkills().skills` |
| `[Prompts]` | `this.session.promptTemplates` |
| `[Extensions]` | `this.session.resourceLoader.getExtensions().extensions` |
| `[Tools]` | `this.session.getAllTools()`（内置 + 扩展工具，标注 `(inactive)` 表示未激活） |

### 资源加载

实际资源文件由 `packages/coding-agent/src/core/resource-loader.ts` 负责加载，扫描以下目录：

- Context: `.pi/agent/AGENTS.md`
- Skills: `.pi/skills/` 目录
- Prompts: `.pi/prompts/` 目录
- Extensions: `.pi/extensions/` 目录 + CLI `--extensions-dir` 指定的路径

## 3. 其他启动输出

### 模型 scope 提示

在 `interactive-mode.ts` 约第 711–722 行：

```ts
console.log(theme.fg("dim", `Model scope: ${modelList}${cycleHint}`));
```

### 快捷键名称动态获取

快捷键名称（如 `ctrl+o`、`escape`、`ctrl+c`）并非硬编码字符串，而是通过 `keyText()` 函数从 keybindings 配置动态读取：

- `packages/coding-agent/src/core/keybindings.ts` — 定义所有默认快捷键映射

```ts
// 例如 ctrl+o：
"app.tools.expand": { defaultKeys: "ctrl+o", description: "Toggle tool output" },
```

### 禁用启动信息

启动时传入 `--quiet` 或 `-q`，或者在设置中启用 `quietStartup`，可跳过除模型 scope 外的所有启动输出。
